# /explain - Code Explanation

Explain `$ARGUMENTS` in detail:

## Explanation Levels

### Overview
- What does this code do?
- Why does it exist?
- How does it fit in the system?

### Detailed
- Line-by-line walkthrough
- Algorithm explanation
- Data flow description

### Context
- Dependencies and relationships
- Usage examples
- Edge cases

## Explanation Format

```
ðŸ“– Explanation: [target]

## Summary
[One paragraph overview]

## How It Works
[Step-by-step explanation]

## Key Concepts
- [Concept 1]: [explanation]
- [Concept 2]: [explanation]

## Data Flow
[How data moves through this code]

## Dependencies
- [Dependency 1]: [why needed]
- [Dependency 2]: [why needed]

## Usage Examples
[Code examples showing usage]

## Gotchas
- [Potential pitfall 1]
- [Potential pitfall 2]
```

## Examples

```
/explain src/auth/jwt.ts
/explain the useEffect hook in Dashboard.tsx
/explain how the payment flow works
/explain this regex pattern: /^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$/
```

## Tips

- Ask for specific aspects if needed
- Request diagrams for complex flows
- Ask follow-up questions for clarity
